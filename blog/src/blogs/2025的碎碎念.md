# 2025的碎碎念

荒唐的一年...

2024年发生了很多很多非常不开心的事情，也因为不清楚自己未来该选择什么样的路焦虑，也谢谢自己和家人朋友们在2025年的时候没有放弃这样没用的破酥吧。

2025 年遇到了好多人，发生了好多事。2024 年末在最痛苦的时候一不小心点开邦邦第一季，成了我邦邦生涯开始的地方。哈哈，永远忘不掉一月18号的那个看完邦o在重庆北站挨冻的那个晚上了。在这天之后就正式加入了现在一起看邦邦的小团体，大伙感觉真的和家人一样，一起犯病，一起噶nsy，一起跑现地看莱芜，一起打游戏，一起随机挑一个群友家过夜哈哈哈。2025年对我来说最幸福最幸运的事情就是遇见了这么一帮傻逼，日子就这么过着，认识这些人也快一年了，感谢你们把我从孤独中拉出来。

![神秘聚会](https://pic1.imgdb.cn/item/694e1e375dc0a5b001d85699.jpg)

2月份我从蔚来离职，到了微信境外产品中心实习，开始了我第一次的大厂实习。感谢腾子愿意捞我，也很感谢朋友们的陪伴与支持，感谢某位群友的大手，也感谢在蔚来的同事们对我的照顾，也很想留在那里，但是列车总会开往下一站，人总得向前走。2024年五月的迷茫与不知所措，六月的下定决心，那时候憧憬过，也怀疑过，但也没过这一天能来的这么快，希望这份好运气继续下去吧。

![](https://pic1.imgdb.cn/item/694e1f157d03703bbdb17b32.jpg)

接下来就是半年的实习生活了，真的是痛苦又快乐着。值得一提的是，在2024年的2月，我在深圳湾公园和几个福建老乡散步，曾经说过，要是我明年能来这里实习就好了，而就在2025年的二月，我走进了腾讯滨海大厦。人生就是这么充满了戏剧性吧，从来没想过一年前的随口一说竟然变成了现实。真的很幸运，同组的实习生很多人干的都是奇奇怪怪的活（比如有个人是后端出身走前端进组进公司在搞数据分析哈哈），我面的前端进来也真的在干前端，mt 也是一个很好很温柔的人，给我分配需求的时候从来都不是直接丢给我，而是和我详细谈论这个需求，问我我的思路是什么，想怎么样去实现它，然后在我的想法上给出很好的建议，非常尊重我自己的想法，而不是只告诉我要做什么。实习期间也谢谢 ld 相信我这个小菜鸡的能力哈哈哈，让我独立负责了一个用于部门提效的 agent 工具落地，这也为我后续的秋招提供了莫大的帮助。

![](https://pic1.imgdb.cn/item/694e210960ffb23a2bf93f62.jpg)

实习期间还跑了好多趟日本看莱芜哈哈哈哈，这里就不做过多叙述了，想看的可以看我另一篇博客：[文章详情 - C4iN's Blog | 破酥的个人博客](https://blog.cainsu.site/#/blogs/Popipa10)。关于我是如何半年学前端的，我这里把我之前写过的一篇文章直接抄过来了。

大一大二时还是一个坚定的保研er，每天都在和密码学几十页没有图片全是完全看不懂的数学符号的纯英文论文（哈哈）。到了大二下的暑假，我才发现自己根本不适合搞科研，大学的前两年单纯是为了迎合家人的期待做的无用功，浪费了不少的时间。现在秋招也快开奖了，回顾一下不经感慨当时直接全身心准备就业该有多好，不过也是前两年的漫无目的地探索让我在最后发现了自己真正喜欢的东西。这里也希望大伙能尽早找到自己真正喜欢且适合自己的的东西吧。博主是个 Vuer，没咋写过 React ，实习的时候干了很多前端之外的事，在这里希望这些学习经验/踩过坑能帮到你。

我是从24年八月开始正式学习前端的，先用一两周快速过了一下前端三件套的基础用法，然后上手搓了一个纯前端的管理系统大概了解了一下 Vue 以及 Vue 相关的工具库，还有 typescript，顺带巩固前端三件套的用法，大概一个月，搞完这个就开始准备写在简历上的项目了。我写在简历上的项目是你室的招新平台、自己搓的一个博客系统以及一个组件库。对于简历项目的准备，需要贯彻的是一个理念：即使你没真的写过代码，但只要你看懂了，并且能说出个所以然，这个项目就是你的了。就拿招新平台举例子，由于我是mmx方向的，没有参与过当时招新平台的开发，就找恩师 ddb 要了招新平台的代码就开啃了。你室的招新平台项目还是做的挺规范的，想要在你简历上添一笔的同学可以好好研究一下。对于入手一个大型项目，我们当然要从 README 入手。项目的基本结构，怎么运行，哪些文件是干啥的，一些基础的开发规范是啥样的，一个好的 README 都可以学到很多东西。大概了解这个项目是咋样的后，就可以开始通过运行这个项目来辅助你阅读这个项目了。我的做法是，一个一个页面慢慢来，先对这个页面有哪些功能有个初步的了解，然后找到这个页面对应的文件，有个基础的代码结构了解，然后再逐个击破，知道各个文件的具体内容后，再把所有东西串联起来。以上是我的个人习惯，其实也是我平时写代码的一个基本思路。

关于这个组件库，其实我是很推荐大伙自己从设计开始做一个的，网上也有很多的成熟代码和设计可以参考。为啥这么说呢，因为组件库这种纯前端的玩意，是很适合巩固和提升你的前端技术的，它不涉及很多的的前后端交互，技术栈聚焦于前端，同时对于理解和使用前端工程化这个概念也有很大的帮助。而且作为一个合格的前端er，你不仅仅技术要好，一个良好的设计头脑和产品思维，以及一个良好的审美都是非常非常重要的，前端是离用户最近的一环，有这些软实力对你的发展也有很大的帮助。

接下来就是准备面试了，这块我不多说，每个人都有各自的方法。我个人来说，首先是八股一定要背熟，对于一个没有实习的人来说，很多情况下面试官对你的项目都不会很感兴趣，大部分问项目都是根据你的项目来问八股。项目就不说了，如果你都没办法说服自己你的项目有多牛逼，那你咋说服工作不知道多少年的面试官呢。接下来一个很重要的部分就是算法，对于前端来说力扣 hot100 是一定要刷熟悉的，算法思路一定是需要深入掌握的，很多时候你写不出来面试官会让你说说思路，有时候说出来做不出来也是能过的。然后剩下的交给运气吧，运气大于实力这一块（

七月低我就离职回家准备秋招了。感觉我的秋招平平无奇哈，字节和腾讯都是我还没打算开始投的时候就约面了，由于实习的时候有过非常对口的经历，也很幸运都通过了。然后我就开摆了享受大学生活去了哈哈哈哈。也说说我是怎么准备秋招的吧。首先我先规划了一下秋招都需要准备啥，然后把每天都要学什么安排的清清楚楚，学完就开始玩电脑。老生常谈，算法+实习+项目+八股四手抓，不过这里要说面试主要还是你的面试官看对不对眼哈哈哈，其他我觉得跟上文提到的都差不多了，只不过涉及的广度和深度会更大一些，这里可以把你的简历喂给 GPT，让他狠狠拷打你）秋招的时候如果你有实习，建议重点准备，博主在面 wxg 的时候把实习每一行都问了一遍，拷打时间长达两个半小时（

2025年好像也就干了这么些事吧。想留的人留不住，想完成的事情无法完成，付出的真心总是被人忽视和践踏，2025 总是遗憾与收获并存，好在为了未来付出的努力终于有了它应该有的回报。我做过很多很多的错事，在深夜一个人听着歌独自哭泣，因为失去彻夜不眠，咬着牙拼命将自己拉出深渊和孤独，也因为芝麻小事满心欢喜。人生总是跌宕起伏，也正是因为这样我们才有了活着的意义吧。
